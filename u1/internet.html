¿Que es Internet?

Red informática mundial, descentralizada, formada por la conexión directa entre computadoras mediante un protocolo especial de comunicación.

Internet se podría definir como una red global de redes de ordenadores cuya finalidad es permitir el intercambio libre de información entre todos sus usuarios.

utilizan la familia de protocolos TCP/IP, lo cual garantiza que las redes físicas heterogéneas que la componen constituyen una red lógica única de alcance mundial. ​Sus orígenes se remontan a 1969, cuando se estableció la primera conexión de computadoras, conocida como ARPANET, entre tres universidades en California (Estados Unidos).

¿Que son los Proveedores de Internet?

es la empresa que brinda conexión a Internet a sus clientes. Un ISP conecta a sus usuarios a Internet a través de diferentes tecnologías como ADSL, cablemódem, GSM, dial-up, fibra óptica, satélite, streaming, etc.

No obstante, las funciones de un proveedor de servicios de Internet no se limitan solo a proporcionar acceso online, sino que también incluyen el mantenimiento de la infraestructura clave para respaldar las comunicaciones digitales y garantizar la conectividad como el cableado, los servidores y las redes.

Por otro lado, como se mencionó antes, algunos ISPs ofrecen también servicios adicionales de hosting, diseño web, ciberseguridad, softwares para e-commerce, cuentas de correo electrónico y servicios de registro de dominios.

¿Protocolos de Aplicación?

Un protocolo de comunicaciones es un sistema de reglas que permiten que dos o más entidades de un sistema de comunicación se comuniquen entre ellas para transmitir información por medio de cualquier tipo de variación de una magnitud física. Existen varios protocolos estandarizados de aplicación:

  • HTTP (páginas web)
  • FTP (transferencia de archivos)
  • Telnet (conexión a servidor)
  • SSH (conexión segura a servidor)
  • SMTP (correo electrónico)
  • POP (correo electrónico)
  • DNS (resolución de nombres de dominio)

¿ Carpeta Local vs Carpeta Remota?

Carpeta Local:

La carpeta local se refiere a los archivos y datos almacenados en tu propio sistema o computadora.

Estos archivos no se sincronizan automáticamente con otros sistemas o dispositivos.

Ejemplos de carpetas locales son las que encuentras en tu disco duro o en tu perfil de usuario en Windows (como Documentos o Descargas).

Los cambios en los archivos locales no afectan a otros sistemas.

Carpeta Remota:

La carpeta remota se encuentra en un servidor o en otro sistema al que puedes acceder a través de la red (como Internet).

Estos archivos pueden sincronizarse con otros sistemas o dispositivos.

Ejemplos de carpetas remotas son las que se utilizan para almacenar archivos en la nube o en servidores de alojamiento web.

Los cambios en los archivos remotos afectan a todos los sistemas que acceden a ellos.

¿Que es la experiencia de usuario?

La experiencia de usuario se refiere a cómo los usuarios interactúan con un producto o servicio. Incluye aspectos como la usabilidad, accesibilidad, diseño visual y emocional, entre otros.

¿Que es la interfaz de usuario ?

La interfaz de usuario es la parte visible con la que los usuarios interactúan en una aplicación o sitio web. Incluye elementos como botones, menús, formularios y diseño visual.

¿Que elementos debo tener en cuenta al contratar un proveedor de hosting para mi proyecto web?

  • Velocidad: La rapidez del servidor es crucial para la experiencia del usuario y el SEO.
  • Confiabilidad: Busca proveedores con alto tiempo de actividad y servicios consistentes.
  • Seguridad: Asegúrate de que ofrezcan certificados SSL y protección contra ataques.
  • Características Adicionales: Evalúa las herramientas y servicios extras que ofrecen.
  • Soporte Técnico: Considera la calidad y disponibilidad del soporte al cliente.

Consideraciones Adicionales

  • Entiende las necesidades específicas de tu proyecto para elegir el tipo de alojamiento adecuado.
  • La ubicación geográfica de los servidores puede influir en los tiempos de carga.
  • Revisa las cláusulas legales del contrato, como la propiedad de dominios y la protección de datos.

¿Que es y para que sirve el html semántico.?

El HTML semántico utiliza etiquetas HTML para dar significado al contenido, facilitando la comprensión tanto para los navegadores como para los desarrolladores. Ejemplos de etiquetas semánticas son <header>, <nav>, <article>, <section>, <footer>, entre otras.
